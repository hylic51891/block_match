# 第三期开发计划（微信小游戏完整项目版）

## 1. 文档目的

本文档用于定义《回路连连看》第三期开发范围、阶段目标、关键里程碑和实施步骤。

第三期的核心目标不是继续扩充本地关卡内容，而是：

> 把项目升级为一个完整的、可在微信小游戏运行的产品雏形。

---

## 2. 第三期总目标

### 2.1 产品目标
- 从“20 关本地原型”切换为“教学关 + 每日挑战”结构
- 构建更适合微信小游戏的碎片化挑战链路
- 为分享、广告、登录、排行榜等平台能力做产品预留

### 2.2 技术目标
- 形成 Web + 微信小游戏双环境方案
- 保持 core 逻辑复用
- 新增小游戏平台适配层
- 重构存档、结果、挑战配置结构

### 2.3 文档目标
- docs 总览全面反映当前新目标
- 产出第三期 PRD
- 产出第三期任务拆分与节奏计划

## 5. 风险与应对

### 风险 1：一边改产品结构，一边改平台，导致范围失控
应对：
- 先完成挑战制产品改造
- 再接微信适配层
- 不同时推进完整后端和完整平台能力

### 风险 2：微信小游戏适配直接污染核心逻辑
应对：
- 所有 wx 相关调用必须进 adapter/service
- core 不允许直接 import wx

### 风险 3：旧存档结构与新结构冲突
应对：
- 加迁移逻辑
- 至少保证旧数据解析失败时可安全重置

### 风险 4：挑战制改造后失去可测试性
应对：
- challenge provider、BattleResult、progress migration 必须补测试
- 保留 Web 环境做自动测试与快速回归

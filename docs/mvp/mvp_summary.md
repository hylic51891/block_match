# 《回路连连看》MVP 总结文档

> 本文档面向产品 AI，用于基于当前实现状态规划下一阶段。

## 一、产品概述

**名称：** 回路连连看
**定位：** 基于"同图块配对消除 + 有限转折路径连接 + 污染封路机制"的轻度休闲益智小游戏
**当前阶段：** MVP 本地试玩版，可在浏览器中运行，核心玩法闭环完整
**技术栈：** React + TypeScript + Vite + Canvas 2D + Zustand + Vitest

---

## 二、核心玩法

### 2.1 基础规则
- 玩家点击两个**相同类型**的图块
- 若两者间存在合法路径（仅水平/垂直移动，最多 **2 次转折**），则消除
- 棋盘全部清空 = 通关
- 无可消除配对 + 无重排次数 = 失败

### 2.2 路径规则
- 路径只能水平或垂直移动
- 最多 2 次转折
- 路径不能穿过：障碍物、其他图块、污染格
- **不支持**棋盘外圈环绕

### 2.3 污染机制（差异化核心）
- 每次成功消除后，路径经过的**空白格**变为污染格
- 污染格在 N 回合内不可被路径穿过
- 污染格随回合推进自动衰减恢复为空白格
- 当前污染持续：2~3 回合（按关卡配置）

### 2.4 灵能图块（特殊图块）
- 带**金色发光边框**的特殊图块
- 两个灵能图块配对时：
  - 可**穿过污染格**（普通图块不行）
  - 连线可多拐**1 次弯**（最多 3 折 vs 普通 2 折）
- 当前有两种灵能图块：**S**（粉红）和 **T**（翠绿），能力相同

### 2.5 重排机制
- 玩家有限次重排机会（随关卡递减）
- 重排会重新分配剩余图块的位置（不改变种类分布）
- 重排后保证至少存在一组可消配对

## 三、关卡设计

### 3.1 20 关总览

| 关卡段 | 棋盘 | 普通种类 | 特殊种类 | 障碍 | 污染 | 重排 | 教学提示 |
|--------|------|----------|----------|------|------|------|----------|
| 1-3 | 6×6 | 4 (A-D) | — | 0 | 关 | 3 | 关1: 基本操作 |
| 4-5 | 6×8 | 5 (A-E) | — | 0 | 关 | 3 | — |
| 6-7 | 8×8 | 5 (A-E) | — | 2~3 | 关 | 3 | — |
| 8-10 | 8×8 | 6 (A-F) | — | 3~4 | 关 | 2 | — |
| 11-12 | 8×8 | 6 (A-F) | — | 2~3 | 开(2回合) | 2 | 关11: 污染机制 |
| 13-14 | 8×8 | 6 (A-F) | S | 3 | 开(2回合) | 2 | 关13: 灵能图块 |
| 15-16 | 10×8 | 6 (A-F) | S | 4 | 开(2回合) | 2→1 | — |
| 17-18 | 10×10 | 7 (A-G) | S / S+T | 5 | 开(2~3回合) | 1 | — |
| 19-20 | 12×10 | 8 (A-H) | S+T | 6 | 开(3回合) | 1 | — |

### 3.2 难度递进设计
- **地图扩大**：6×6 → 6×8 → 8×8 → 10×8 → 10×10 → 12×10
- **种类增加**：4 → 5 → 6 → 7 → 8 种普通图块
- **机制引入**：障碍 → 污染 → 灵能图块
- **容错递减**：重排次数 3 → 2 → 1
- **污染加重**：持续 2 回合 → 3 回合

### 3.3 棋盘填满规则
所有关卡 `fillBoard: true`：除障碍物位置外，所有格子均被图块填满。图块按种类均匀分配，特殊图块每种出现 1 对（2 个）。

### 4.2 已知问题与待完善 ⚠️

| 项目 | 当前状态 | 影响 |
|------|----------|------|
| **T 类型灵能图块** | 关卡中已使用，但与 S 能力完全相同，无差异化 | 机制单调 |
| **消除动画** | `removing` 状态已定义但未使用，消除无过渡动画 | 体验生硬 |
| **路径动画** | 路径瞬间出现并消失，无绘制动画 | 视觉反馈弱 |
| **星级评价** | bestStar 只有 0/1，无 2/3 星 | 无重玩激励 |
| **计分系统** | score = matchCount，无连击/时间加成 | 无竞争感 |
| **教程系统** | 仅 3 关有 tutorial 文本，无交互式引导 | 新手体验差 |
| **关卡解锁** | 所有关卡全部开放，无前置解锁 | 无进度感 |
| **图块视觉** | 仅颜色+字母区分，无形状/图标 | 色盲不友好 |
| **路径失败原因** | 定义了 6 种细粒度原因但总是返回 `no_path` | 无法做智能提示 |
| **棋盘外圈通道** | 不支持路径绕出棋盘外 | 限制策略空间 |
| **音效** | 无 | 缺少反馈层次 |
| **关卡配置** | 障碍位置硬编码，无程序化生成 | 难以扩展 |

### 4.3 架构分层

```text
src/
  app/          # 入口、场景路由
  scenes/       # HomeScene / BattleScene / ResultScene
  core/
    board/      # 棋盘模型 + 生成器
    level/      # 关卡加载 + 20关数据
    rules/      # PathFinder + MatchResolver
    systems/    # DeadlockDetector / ShuffleManager / PollutionSystem / WinLoseChecker

  render/       # Canvas 渲染器
  store/        # Zustand 状态管理 + 存档 + 埋点
  infra/        # storage / logger / telemetry / error-reporter
  hooks/        # useCanvas / useClickHandler / useGameLoop
  components/   # BattleHUD / LevelCard / DebugPanel
  types/        # 全局类型定义
  assets/       # 颜色配置
  tests/        # 45 个测试用例
```

## 七、下一步规划建议

以下为产品 AI 规划时的参考方向：

1. **差异化灵能图块**：S 穿污染+多一折，T 可定义新能力（如：消除时不产生污染 / 可穿越一个图块 / 消除时清除周围污染）
2. **动画系统**：消除闪烁、路径绘制动画、污染淡入淡出
3. **星级评价**：根据回合数/重排次数/用时评 1~3 星
4. **关卡解锁**：前一关通关解锁下一关
5. **交互式教程**：高亮可点击图块、箭头指引
6. **障碍物扩展**：可破坏障碍（需消除相邻图块）、冰冻图块（需匹配2次才消除）
7. **棋盘外圈通道**：允许路径绕出棋盘边界
8. **程序化关卡生成**：基于难度参数自动生成关卡配置
9. **音效**：点击、消除、污染、重排、通关、失败
10. **微信小游戏迁移**：平台适配、登录、分享、广告位预留
